# 中国期货市场

集合竞价

1. 按照orderbook在不同价位累加bid volume和ask volume

2. 在不同价位计算trade volume = max(bid volume, ask volume)
3. 选择trade volume最大,而remain volume最小的价位作为成交价

期货结算

- 用最后5分钟的VWAP来作为结算价,既能减少价格被操纵的可能,又能及时反映市场
- 对于空头和多头,保证金比例相同,且券商对保证金的要求比交易所更高
- 当连续3天出现涨跌停极端行情时,交易所会强制平一些仓位

# 基本算法

### 套利

- 利润来源: 价差稳定,但是有少量波动
- 风险:汇率/合约单位/价格偏离/转运/<u>套利交易时点</u>
- 套利类型:跨期/跨境/跨交易所/跨品种
- 操作逻辑
  1. 重新定义虚拟交易对的买卖操作/价格和交易量
  2. 使用重新定义的交易对来分析利润关系
  3. 在交易时,优先交易盘口宽的标的, 成功后再交易另一个

### 量价关系分析

- 从量价信息中分析市场参与者的特征
- 只使用最近数据来提取信息,因为市场参与者会缓慢变化

### 做市商

- 需要区分交易对手方是否携带信息
- 为市场提供流动性赚取盘口差价

- 封装交易接口,和交易所/券商API对接,并给一些特殊订单提供封装

  | Changing Order       | FAK Order                      | Market Order                      | Tentative Order                                          | Mandatory Order          |
  | -------------------- | ------------------------------ | --------------------------------- | -------------------------------------------------------- | ------------------------ |
  | 撤单并立刻提交新订单 | 提交订单并立刻撤单(如果没成交) | 提交限价单,如果没有全部成交就撤单 | 在指定价位提交订单,如果没有满足条件(超时/价格变化)就撤销 | 必须完成一定交易量的订单 |

- 模拟回测功能,使用市场行情数据来模拟交易

- 风险控制

  | 仓位控制                  | 订单限制       | 资金限制   | 价格限制             | 交易限制                | 流量限制                           |
  | ------------------------- | -------------- | ---------- | -------------------- | ----------------------- | ---------------------------------- |

- 监控系统: 和交易系统分离的线程/进程,负责IO且不会影响交易系统效率

- 操作系统优化
